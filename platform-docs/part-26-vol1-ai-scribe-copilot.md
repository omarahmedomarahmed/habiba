---

## AI Philosophy

The AI should **never replace** therapists.  
The AI should **amplify** therapists.

The AI should help therapists:
- Listen better
- Document faster
- Remember more
- Organize information
- Track progress
- Reduce administrative burden

The AI should **not** independently:
- Diagnose patients
- Prescribe medications
- Override clinical judgment
- Make treatment decisions without clinician review

Clinical recommendations should always be presented as **suggestions for professional review**.

## AI System 1: Realtime AI Scribe Engine

**The flagship product.**

**Goal:** Listen continuously during sessions.

**Inputs:**
- Audio
- Video Audio
- Phone Calls
- Live Meetings
- Uploaded Recordings

**Outputs:**
- Transcript
- Session Notes
- Suggested Questions
- Session Summary
- Action Items
- Clinical Observations
- Structured Data

### Live Session Pipeline

```
Audio Stream
      ↓
Speech Recognition
      ↓
Speaker Identification
      ↓
Realtime Transcript
      ↓
Clinical Understanding
      ↓
Copilot Suggestions
      ↓
Memory Updates
```

**Target latency:** 1–3 seconds

> The therapist should feel like the AI is listening alongside them.

### Speaker Identification

System identifies:
- **Therapist**
- **Patient**
- **Family Member**
- **Interpreter**

Every transcript segment is labeled.

### Transcript Enrichment

Raw transcripts have limited value. The system continuously extracts:
- Topics
- Symptoms
- Goals
- Stressors
- Life Events
- Relationships
- Medication Mentions
- Behavioral Signals
- Assessments
- Risk References

## Realtime Session Intelligence

As the conversation unfolds, AI builds understanding.

**Example:**

Patient says: *"I haven't been sleeping much lately."*  
AI recognizes: **Potential Sleep Disturbance**

Patient later says: *"I've also been missing work."*  
AI connects: **Sleep Issues → Functional Impairment**

This becomes **structured memory**.

## AI System 2: Therapist Copilot

**One of the most valuable features.**

Appears alongside session. Never interrupts. Never takes over.

Provides:
- Suggestions
- Insights
- Reminders
- References
- Questions

### Copilot Panel (Live Side Panel)

Sections:
1. **Conversation Summary** — running real-time summary
2. **Suggested Follow-Up Questions**
3. **Potential Themes** — detected topics
4. **Goals Mentioned** — referenced treatment goals
5. **Medication Mentions** — any drug references
6. **Risk Review Reminders** — if risk language detected
7. **Session Timeline** — what happened and when

### Question Suggestion Engine

As discussion evolves, AI proposes follow-up questions.

**Example:**

> Patient: "I feel anxious every Sunday."  
> Copilot suggests: "Would you like to explore what typically happens before Sunday evenings?"

Presented as optional guidance.

## AI System 3: Clinical Memory Engine

**This becomes the most valuable data asset.**

Most EHRs store **documents**.  
24Therapy stores **understanding**.

**Traditional system:**
> "Session Note #43"

**24Therapy:**
> "Patient consistently reports increased anxiety after workplace conflicts."

Structured. Searchable. Trackable.

### Memory Types

| Type | Description |
|------|-------------|
| Symptom Memory | Persistent symptom patterns |
| Goal Memory | Active and completed therapy goals |
| Relationship Memory | Key people in patient's life |
| Life Event Memory | Major events impacting therapy |
| Medication Memory | Medication history and changes |
| Risk Memory | Historical risk indicators |
| Treatment Memory | Interventions used |
| Strength Memory | Patient strengths and resources |
| Protective Factor Memory | Factors promoting resilience |

### Memory Confidence System

Every memory receives:
- **Confidence Score** (0–1)
- **Source References** (session IDs, segments)
- **Review Status** (pending/approved/rejected)

Therapist can:
- **Approve** — confirm memory is accurate
- **Modify** — correct details
- **Reject** — discard incorrect extraction

### Memory Evolution

Memory changes over time:

```
Sleep Issues
  Mild → Moderate → Severe → Improving
```

Tracked historically with full timeline.
